The new service’s README was comprehensive, formatted with precision, and generated by an agent that had scanned the entire codebase. The Staff Engineer, reviewing the pull request, found a five-page document describing every function and variable in the system with balanced, eerie prose. However, during a production incident two weeks later, the on-call engineer found the document useless. It was technically accurate but lacked salience—the human-derived hierarchy of what matters when the system is failing. It was a map of the territory that failed to mention where the mines were buried.

## The Rise of Machine-to-Machine Artifacts

Engineering organizations are adopting tools that optimize for context density and token efficiency. Proxies compress code into images to save on API costs, while agents summarize hundreds of Slack messages into a single paragraph. These are rational responses to the pricing structures of frontier intelligence; if tokens are the primary cost center, reducing token count is a high-leverage task.

The result is a class of Machine-to-Machine (M2M) documentation and code—artifacts legible to an LLM but requiring significant cognitive effort for a human to parse. In a QBR, a Director can point to a 60% reduction in token costs or a 100% increase in documentation coverage. These green metrics hide a Comprehension Tax: the invisible delay added to every human interaction with the system.

## The Salience Gap

A human writing a README makes thousands of judgment calls about what to omit, emphasizing the "why" and the historical failures. This provides a thought hierarchy that guides the reader’s attention. An AI agent, optimized for coverage, often provides a flat landscape of information. It treats the initialization of a logger with the same weight as the logic for a distributed lock. To a machine, this is just data; to a human, it is noise.

The Comprehension Tax is paid during critical moments: onboarding, where a new hire must dig through mountains of AI-generated prose to find the core mental model, and incidents, where the lack of clear hierarchy in an AI-summarized log prevents the engineer from entering a flow state of diagnosis.

As the cost of producing text drops, the value of editing—the act of subtraction and emphasis—becomes the primary bottleneck. Organizations that treat AI-generated documentation as a set-and-forget asset may find their organizational legibility decreasing, even as their data coverage increases. The most expensive documentation is not the one that costs the most tokens; it is the one that costs the most minutes of a Staff Engineer’s attention.
