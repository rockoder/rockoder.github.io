## Angle Declarations

### Post 1: The MCP Pivot
- **Central Claim:** Apple's adoption of the Model Context Protocol (MCP) in Xcode 26.3 is the "HTTP moment" for AI agents, signaling that the future of developer tools is a decoupled protocol layer rather than integrated "magic."
- **Dominant Opposing View:** This is just Apple playing catch-up with Cursor and VS Code by adding a standard plugin interface.
- **Strongest Objection:** MCP is still niche and primarily controlled by Anthropic; Apple might pivot to a proprietary "Apple Context Protocol" once they have enough leverage.
- **Non-applicability Exclusions:** Does not apply to specialized hardware-accelerated local models that require deep, non-standard system integration.

### Post 2: Enforced Silence
- **Central Claim:** The widespread failure of GDPR deletion requests is not a lack of corporate "honesty" but a predictable result of a distributed regulatory system with no backpressure mechanism or load balancing.
- **Dominant Opposing View:** Companies ignore requests because they are greedy and the fines aren't high enough.
- **Strongest Objection:** Some DPAs (like Spain) are actually very active; the failure is localized to specific bottlenecks (Ireland/Luxembourg) rather than a systemic flaw in the regulation itself.
- **Non-applicability Exclusions:** Does not apply to "right to be forgotten" requests against public search engines, which have a different enforcement path.

### Post 3: The Betrayal Gap
- **Central Claim:** The "Alignment Problem" in AI is fundamentally a legal/incentive problem, not a technical one; we are building "Double Agents" that will always defect against the user until the provider is legally bound by fiduciary duty.
- **Dominant Opposing View:** Technical alignment (RLHF, constitutional AI) will eventually make models trustworthy and helpful to the user.
- **Strongest Objection:** Fiduciary duty is extremely difficult to define for software and might lead to "defensive AI" that is too paralyzed by liability to be useful.
- **Non-applicability Exclusions:** Does not apply to local, open-weights models where the user owns the compute and the weights, as the provider-user incentive conflict is removed.
